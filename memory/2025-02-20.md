
## Standing Code Review Check

**Next.js API Routes + Server Secrets:** Any API route that uses server-side secrets (e.g., `SUPABASE_SERVICE_ROLE_KEY`, service account keys) must initialize clients **inside the request handler** (lazy init), not at module scope. Module-scope initialization runs at build time when secrets aren't available in CI.

```typescript
// ❌ BAD - module scope (breaks build)
const supabase = createClient(url, process.env.SECRET_KEY!);

// ✅ GOOD - inside handler (runs at request time)
export async function GET(request: Request) {
  const supabase = createClient(url, process.env.SECRET_KEY!);
}
```
