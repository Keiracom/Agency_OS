## Key Technical Lessons

### Clean slate FK order
Must delete in this order to avoid FK violations:
1. `leads` (has FK to lead_pool, campaigns)
2. `lead_pool` (has FK to campaigns)
3. `campaigns`

### asyncpg datetime rule
- Columns mapped as `Mapped[datetime]` (no timezone) → use `datetime.now(UTC).replace(tzinfo=None)`
- Columns mapped as `Mapped[datetime]` with `DateTime(timezone=True)` → `datetime.now(UTC)` is fine
- `ConversionPattern.valid_until` is tz-naive despite DB being `TIMESTAMP WITH TIME ZONE`

### SQLAlchemy ORM update() attr names
`update(Lead).values()` must use Python attribute names, NOT DB column names.
Lead model has mismatched names (propensity_* Python → als_* DB):
- `propensity_tier` → DB: `als_tier`
- `propensity_data_quality` → DB: `als_data_quality`
- `propensity_authority` → DB: `als_authority`
- `propensity_company_fit` → DB: `als_company_fit`
- `propensity_timing` → DB: `als_timing`
- `propensity_risk` → DB: `als_risk`

### ConversionPattern model
Inherits `TimestampMixin` (NOT SoftDeleteMixin). No `deleted_at` column.
