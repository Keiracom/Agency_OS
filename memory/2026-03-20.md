### Key technical learnings
- BD snapshot: must wait ~30s after "ready" before downloading (assembly lag)
- Bulk REST upsert to Supabase (`/rest/v1/table?on_conflict=abn`) = 3s for 1,000 rows
- MCP bridge SQL truncates large result sets — use Supabase REST API directly for data queries
- Supabase REST returns 201 (not 200) for POST/upsert — treat 201 as success
- BD error records have `input.keyword` not `discovery_input.keyword`
- NDJSON download from BD requires line-by-line `json.loads()`, not `json.loads(full_text)`
