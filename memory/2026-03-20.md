### Critical technical learnings
- **Exec SIGTERM pattern**: OpenClaw kills running exec sessions when processing heartbeat/queued messages (~280s hard limit OR immediately on new message arrival if yieldMs was used)
- **Solution**: No yieldMs, no background, strict timeout=200s, 3 prefixes per call (~130-170s)
- **pgbouncer**: DATABASE_URL uses pgbouncer transaction mode → `asyncpg.connect(..., statement_cache_size=0)` required
- **curl 2s timeout**: Supabase does NOT start DB query if HTTP connection drops before full request delivery (tested: only prefix 18 fired before full loop — 2s was enough for one but not rapid-fire 82)
- **MCP bridge per-prefix**: ~65s per prefix via Management API (not 30s as expected from small batches)
- **Parallel spray**: 90 concurrent curl -m 2 requests fired but DB only processed ~8 before connections dropped — rate limited or timing dependent
- **asyncpg deadlock**: Happened when spray + sequential ran concurrently on same table — pg_cancel_backend resolved it
