## Key Decisions

| Decision | Rationale |
|----------|-----------|
| **Personas created by platform, not clients** | Instant Day 1 outreach - no waiting for client setup |
| **LinkedIn only client-provided asset** | Everything else (domains, mailboxes, warmup) handled by platform |
| **40% buffer rule** | Event-driven replenishment triggered by Stripe signup webhook |
| **Daily warmup polling (not webhooks)** | WarmForge has no webhooks - daily cron at 6am AEST checks heat scores |
| **Heat score ≥85 = AVAILABLE** | WarmForge threshold for marking domains ready to send |

## Salesforge API Notes

- **Auth:** `Authorization: {api_key}` header (plain key, NOT Bearer)
- **Base URL:** `https://api.salesforge.ai/public/v2`
- **Webhooks available:** email_sent, email_opened, link_clicked, email_replied, email_bounced, positive_reply, negative_reply (NOT warmup completion)

## WarmForge API Notes

- **Auth:** `Authorization: {api_key}` header (plain key)
- **Base URL:** `https://api.warmforge.ai/public/v1`
- **Key fields:** `warm: bool`, `warmupDaysCompleted`, `healthReport.heatScore`
- **No webhooks** - must poll for status
